# Design Tokens Reference

## Overview

Our design token system provides a comprehensive set of values that ensure consistency across all components. These tokens are easily customizable by your design team and automatically adapt to light and dark themes.

## Token Categories

### Spacing & Sizing Tokens

Our token system provides both **semantic names** (recommended) and numeric values for flexibility:

#### Semantic Tokens (Recommended)
```typescript
// Spacing tokens - more intuitive and readable
$xs = 4px      // Extra small spacing
$sm = 8px      // Small spacing  
$md = 16px     // Medium spacing
$true = 16px   // Default spacing (same as $md)
$lg = 24px     // Large spacing
$xl = 32px     // Extra large spacing
$2xl = 48px    // 2x Extra large spacing
$3xl = 64px    // 3x Extra large spacing
$4xl = 96px    // 4x Extra large spacing

// Size tokens for component dimensions
$xs = 16px     // Extra small size
$sm = 24px     // Small size
$md = 32px     // Medium size
$true = 32px   // Default size (same as $md)
$lg = 48px     // Large size
$xl = 64px     // Extra large size
$2xl = 96px    // 2x Extra large size
$3xl = 128px   // 3x Extra large size
$4xl = 192px   // 4x Extra large size

// Negative spacing for margin adjustments
$-xs = -4px    $-sm = -8px    $-md = -16px   $-lg = -24px   $-xl = -32px
```

> **Note**: The `$true` token is required by Tamagui as the default baseline value. It allows Tamagui to scale components up or down from this default without making assumptions about your token naming.

#### Numeric Tokens (Available for compatibility)
```typescript
// Fine-grained spacing (2px increments)
$0.25 = 2px    $0.5 = 4px     $0.75 = 6px

// Standard spacing scale (8px base unit)
$1 = 8px       $2 = 16px      $3 = 24px      $4 = 32px
$5 = 40px      $6 = 48px      $8 = 64px      $10 = 80px
$12 = 96px     $16 = 128px    $20 = 160px    $24 = 192px
```


#### Semantic Colors
```typescript
// Success colors (green scale)
$success50 to $success900

// Warning colors (amber scale)  
$warning50 to $warning900

// Error colors (red scale)
$error50 to $error900

// Neutral colors (gray scale)
$neutral50 to $neutral950
```

#### Theme-Aware Colors
These colors automatically adapt to light/dark themes:

```typescript
// Background colors
$background           // Primary background
$backgroundHover      // Hover state background
$backgroundPress      // Press state background
$backgroundFocus      // Focus state background
$backgroundStrong     // High contrast background

// Text colors
$color               // Primary text color
$colorHover          // Hover state text
$colorPress          // Press state text
$colorFocus          // Focus state text

// Border colors
$borderColor         // Default border
$borderColorHover    // Hover border
$borderColorFocus    // Focus border (usually brand)
$borderColorPress    // Press border

// Semantic theme colors
$brand              // Primary brand (theme-aware)
$success            // Success state (theme-aware)
$warning            // Warning state (theme-aware)
$error              // Error state (theme-aware)
```

## Usage Examples

### Component Styling

#### Preferred Approach (Semantic Tokens)
```tsx
// Using semantic design tokens - more readable and maintainable
<Card
  padding="$xl"                    // 32px padding
  margin="$md"                     // 16px margin
  borderRadius="$lg"               // 12px border radius
  backgroundColor="$background"
  borderColor="$borderColor"
  elevation={2}
>
  <Button
    variant="primary"
    size="medium"
    backgroundColor="$brand"       // Theme-aware brand color
    borderRadius="$md"             // 8px border radius
    paddingHorizontal="$xl"        // 32px horizontal padding
    paddingVertical="$md"          // 16px vertical padding
  >
    Click me
  </Button>
</Card>
```

#### Alternative Approach (Numeric Tokens)
```tsx
// Using numeric tokens - still valid but less intuitive
<Card
  padding="$4"              // 32px padding
  margin="$2"               // 16px margin
  borderRadius="$6"         // 12px border radius
  backgroundColor="$background"
  borderColor="$borderColor"
  elevation={2}
>
  <Button
    variant="primary"
    size="medium"
    backgroundColor="$brand"   // Theme-aware brand color
    borderRadius="$6"         // 12px border radius
    paddingHorizontal="$4"    // 32px horizontal padding
    paddingVertical="$2.5"    // 20px vertical padding
  >
    Click me
  </Button>
</Card>
```

### Responsive Spacing

#### Semantic Approach (Recommended)
```tsx
// Using semantic tokens for responsive design
<YStack
  padding="$md"      // 16px on mobile
  $gtSm={{
    padding: "$xl"   // 32px on tablet+
  }}
  $gtMd={{
    padding: "$2xl"  // 48px on desktop+
  }}
>
  Content
</YStack>
```

#### Numeric Approach (Alternative)
```tsx
// Using numeric tokens for responsive design
<YStack
  padding="$2"     // 16px on mobile
  $gtSm={{
    padding: "$4"  // 32px on tablet+
  }}
  $gtMd={{
    padding: "$6"  // 48px on desktop+
  }}
>
  Content
</YStack>
```

### Theme-Aware Styling

```tsx
// Colors that automatically adapt to light/dark theme
<XStack
  backgroundColor="$background"      // White in light, dark in dark theme
  borderColor="$borderColor"         // Light gray in light, dark gray in dark
  padding="$xl"                      // Using semantic spacing token
  borderRadius="$md"                 // Using semantic radius token
>
  <Text color="$color">             {/* Black in light, white in dark */}
    This text adapts to the theme
  </Text>
</XStack>
```

## Customization Guide

### For Design Teams

To customize the design system, modify the token values in `tamagui.config.ts`:

```typescript
const tokens = createTokens({
  // Update brand colors to match your brand
  color: {
    brand500: '#your-primary-color',    // Your primary brand color
    brand600: '#your-hover-color',      // Slightly darker for hover
    brand700: '#your-press-color',      // Even darker for press
    
    // Update semantic colors
    success500: '#your-success-color',
    warning500: '#your-warning-color', 
    error500: '#your-error-color',
    
    // Customize neutral grays
    neutral50: '#your-lightest-gray',
    neutral500: '#your-medium-gray',
    neutral900: '#your-darkest-gray',
  },
  
  // Adjust spacing scale if needed
  space: {
    // Add custom spacing values
    xs: 4,     // $xs = 4px
    sm: 8,     // $sm = 8px  
    md: 16,    // $md = 16px
    lg: 24,    // $lg = 24px
    xl: 32,    // $xl = 32px
  },
  
  // Customize border radius
  radius: {
    none: 0,
    sm: 2,
    md: 4, 
    lg: 8,
    xl: 12,
    full: 9999,
  },
})
```

### Best Practices

1. **Prefer Semantic Tokens**: Use `$sm`, `$md`, `$lg` instead of `$1`, `$2`, `$3` for better readability
2. **Use Theme-Aware Colors**: Prefer `$brand`, `$success` over specific values like `$brand500`
3. **Stick to the Scale**: Use defined tokens rather than arbitrary values
4. **Consistent Naming**: Follow the xs/sm/md/lg/xl convention across spacing, sizing, and radius
5. **Theme Awareness**: Use theme-aware colors (`$background`, `$color`) for automatic dark mode support
6. **Documentation**: Document any custom token additions or modifications

#### Token Selection Guide
```tsx
// ✅ Preferred - Semantic tokens
<Button
  padding="$lg"           // Clear intent: large padding
  borderRadius="$md"      // Clear intent: medium radius
  fontSize="$lg"          // Clear intent: large text
/>

// ✅ Acceptable - Numeric tokens for fine control
<Button
  padding="$3"            // When you need specific numeric control
  borderRadius="$4"       // When semantic doesn't fit
  fontSize="$4"           // When you need precision
/>

// ❌ Avoid - Hardcoded values
<Button
  padding={24}            // No design system reference
  borderRadius={8}        // Hard to maintain
  fontSize={18}           // Not scalable
/>
```

## Migration from Hardcoded Values

### Before (Hardcoded)
```tsx
<Button
  padding={24}
  backgroundColor="#0ea5e9"
  borderRadius={8}
  color="#ffffff"
/>
```

### After (Semantic Design Tokens)
```tsx
<Button
  padding="$lg"              // 24px - semantic and clear
  backgroundColor="$brand"    // Theme-aware brand color
  borderRadius="$md"          // 8px - medium radius
  color="$backgroundStrong"   // Theme-aware high contrast
/>
```

### Alternative (Numeric Design Tokens)
```tsx
<Button
  padding="$3"               // 24px - numeric reference
  backgroundColor="$brand"   // Theme-aware brand color
  borderRadius="$4"          // 8px - numeric reference
  color="$backgroundStrong"  // Theme-aware high contrast
/>
```

The token-based approach provides:
- **Consistency**: All components use the same values
- **Maintainability**: Change once, update everywhere
- **Theme Support**: Automatic light/dark mode adaptation
- **Design System**: Clear hierarchy and relationships between values
- **Readability**: Semantic tokens make intent clearer (when using sm/md/lg)
